HTML

  • The "head" element contains information about the webpage that only the web browser sees.
  • The "body" element represents the visible content shown to the user.
  • The "meta" element is a child element of the "head" parent element and is ussed to specify character set, page description, keywords, author of the document, and viewport setting.
  • The "img" element is used to embed an image in an HTML page.
  • The "a" (anchor) element creates a hyperlink. The enclosed "href" attribute indicates the link's URL destination.
  • The "h1"-"h6" tags define HTML headings and their respective boldness and categorization hierarchy.
  • The "br" tag inserts a line break within the body of the page to create space between paragraphs or sections, and has no end tag.
  • All of this information and much more is available at: W3Schools' HTML Reference Page.

CSS

  • CSS can be applied using three methods: Inline, internal, or external.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines which portion of HTML to which a given style should be applied.
  • A style declaration contains a property and corresponding value.
  • The "display: block" value assigns each image to start on a new line.
  • The "*" is a wildcard which means "all".
  • To link an external CSS style sheet to an HTML page, the "link" element must be inserted into the "head" element after the final "meta" element and before the "title" element".
  • More info about CSS properties/values is available on the: W3Schools CSS Reference Page.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone SSH-value: clones a repo from GitHub onto local computer
  • git add -A: adds all changes to the staging area for next commit
  • git commit -m "desc of what commit contains, written as imperative command in present tense": commits changes to branch
  • git pull origin main: pulls the main branch from repo onto local computer to ensure working with latest version
  • git push origin feature/code_name: pushes local changes to the remote branch in GitHub
  • All of this information and much more about CLI Git commands is available on the: Git Cheat Sheet.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a variable that contains multiple values.
  • A function tells the computer how to perform a certain task.
  • A function must be first defined and consequently called in the Control Flow.
  • A FOR loop is useful for large sets of values and analyzes each value to see if it meets a set of conditions.
  • An IF conditional allows the computer to determine which code to run based on a defined set of criteria.
  • Math.random() is a special function that generates a random number.
